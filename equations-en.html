Hierarchical Equations

A hierarchical equation is an equation written using rank–degree derivatives \(D_r^n\). The main goal is structural simplification: equations that are highly nonlinear in one representation may become separable or closer to linear after a suitable hierarchical lift.

1) Definition

A rank–degree hierarchical equation is any relation involving a function \(f\) and a finite set of its hierarchical derivatives \(D_r^n f\) (possibly with different ranks and degrees).

\[ \mathcal{E}\Big( x,\ f,\ \{D_{r_i}^{\,n_i}f\}_{i=1}^{m} \Big)=0. \]

If you prefer a “max rank” view: a level-\(k\) equation is an equation that uses ranks up to \(k\):

\[ \mathcal{E}_k\big(x,\ f,\ D_{0}^{\bullet}f,\ D_{1}^{\bullet}f,\ \dots,\ D_{k}^{\bullet}f\big)=0. \]

where \(D_{r}^{\bullet}f\) means “some degrees of rank \(r\)” (e.g., degree 1 and degree 2).

Consistency
On this site the rank-\(r\) operator is always defined by applying \(\ln^{(r)}\) on both \(x\) and \(f(x)\): \[ D_{r}^{1} f(x)=\frac{d\,\ln^{(r)}(f(x))}{d\,\ln^{(r)}(x)}. \] This prevents notation drift between pages.

4) Quick classification (by rank)

Rank 0 — Differential (D0)

\[ F\big(x,f,D_0 f,\dots\big)=0 \]
Base structure: additive change in \(x\).

Rank 1 — Relative (D1)

\[ F\big(x,f,D_1 f,\dots\big)=0 \]
Base structure: ratios / scaling in \(\ln x\).

Rank 2 — Logarithmic

\[ F\big(x,f,D_2 f,\dots\big)=0 \]
Base structure: iterated log scales \(\ln(\ln x)\).

Rank 3+ — Higher hierarchy

\[ F\big(x,f,D_3 f,\dots\big)=0 \]
Base structure: nested layers beyond \(\ln(\ln(\ln(\cdot)))\).

2) Notation (D0, D1, and the general form \(D_r^n\))

Your site uses the symbols D0 (classical differential derivative) and D1 (relative derivative). In rank–degree form:

\[ D_0 \equiv D_{0}^{1}, \qquad D_1 \equiv D_{1}^{1}. \]

The strict rank-\(r\) definition (degree 1) is:

\[ \boxed{ D_{r}^{1} f(x)=\frac{d\,\ln^{(r)}(f(x))}{d\,\ln^{(r)}(x)} } \]

Special cases:

\[ D_{0}^{1} f=\frac{df}{dx}, \qquad D_{1}^{1} f=\frac{d\ln f}{d\ln x}=\frac{x}{f}\frac{df}{dx}, \qquad D_{2}^{1} f=\frac{d\ln(\ln f)}{d\ln(\ln x)}. \]
Key idea
The same equation can be written in different ranks. Choosing the right rank often reveals a simpler structure. This is the role of hierarchical lifting.

3) Domain assumptions (avoid hidden singularities)

Because higher ranks use iterated logarithms, every equation should state its valid domain. Here is a practical checklist:

  • Rank 0: usually no extra condition beyond the classical domain of \(f\).
  • Rank 1: need \(x>0\) and \(f(x)>0\) if \(\ln x\) and \(\ln f\) are used.
  • Rank 2: need \(\ln x>0\) and \(\ln f>0\) (often written as \(x>1\) and \(f(x)>1\)).
  • Avoid points where the denominator scale vanishes (e.g., \(\ln x=0\) when differentiating w.r.t. \(\ln(\ln x)\)).
Reviewer-style note: When using \(\ln(\ln x)\) or \(\ln(\ln f)\), explicitly state the stronger conditions \(x>1\) and \(f(x)>1\) (or the equivalent inequalities).

5) Canonical examples (with solved forms)

These examples show how an equation looks at different ranks, and how the solution changes naturally with the rank.

Example A — Rank 0 (classical): \(D_0 f=g(x)\)

\[ D_0 f(x)=g(x) \quad\Longleftrightarrow\quad \frac{df}{dx}=g(x) \quad\Longrightarrow\quad f(x)=\int g(x)\,dx + C. \]

Example B — Rank 1 (relative): \(D_1 f=g(x)\)

\[ D_1 f(x)=g(x) \quad\Longleftrightarrow\quad \frac{d\ln f}{d\ln x}=g(x). \]
\[ \frac{d\ln f}{dx}=\frac{g(x)}{x} \quad\Longrightarrow\quad \ln f(x)=\int \frac{g(x)}{x}\,dx + C \quad\Longrightarrow\quad f(x)=C_0\exp\!\left(\int \frac{g(x)}{x}\,dx\right). \]

Notice the natural multiplicative form of the solution (this is why rank 1 matches power-like scaling).

Example C — Rank 1 logistic-type: \(D_1 f = a - b f\)

This is structurally simple in rank 1 because it becomes separable in \(\ln f\):

\[ \frac{d\ln f}{d\ln x} = a-bf \quad\Longleftrightarrow\quad \frac{1}{f}\frac{df}{d\ln x}=a-bf. \]
\[ \frac{df}{f(a-bf)}=d\ln x. \]

(You can keep it as an implicit solution here, or present the explicit form on the solved-equations page.)

Example D — Rank 2: \(D_2 f=g(x)\)

\[ D_2 f(x)=g(x) \quad\Longleftrightarrow\quad \frac{d\ln(\ln f)}{d\ln(\ln x)}=g(x). \]
\[ \ln(\ln f)=\int g(x)\,d\ln(\ln x) + C \quad\Longrightarrow\quad \ln f = C_1 \exp\!\left(\int g(x)\,d\ln(\ln x)\right) \quad\Longrightarrow\quad f=\exp\!\Big( C_1 \exp(\cdots)\Big). \]
Interpretation
Rank 2 naturally produces “double exponential / nested log” behavior because the unknown becomes \(\ln(\ln f)\). This is exactly what you want when the phenomenon lives on nested scales.

6) Hierarchical lifting (the main method)

Lifting means rewriting the same problem in a higher-rank variable to simplify the structure. A standard lift is:

\[ u=\ln^{(k)}(f) \quad\Rightarrow\quad D_k f = \frac{du}{d\ln^{(k)}(x)}. \]
\[ \mathcal{E}_k\big(x,f,D_0 f,\dots,D_k f\big)=0 \quad \xrightarrow{\;\text{Lift}\;} \widetilde{\mathcal{E}}_{k}\big(\ln^{(k)}x,\ u,\ \tfrac{du}{d\ln^{(k)}x}\big)=0. \]
Main message
The goal is not to “replace calculus”, but to choose the level where the equation matches the natural scaling, so the solver sees a simpler pattern (separable / linear / constant-coefficient, etc.).

7) Practical workflow (mini-solver)

  • Step 1: Identify the growth type (power? exponential? nested? multi-scale?).
  • Step 2: Choose a rank \(k\) that matches the scale (0, 1, 2, ...).
  • Step 3: Transform the unknown: \(u=\ln^{(k)}(f)\).
  • Step 4: Rewrite the equation in \(u\) and \(\ln^{(k)}(x)\).
  • Step 5: Solve in \(u\) using classical methods (separable/linear/etc.).
  • Step 6: Invert the transform (iterated exponentials) to recover \(f\).
  • Step 7: State the domain where all logs are valid.
